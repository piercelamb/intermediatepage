The SnappyData Platform

The SnappyData Platform is a Spark based, in-memory unified cluster that provides streaming analytics, OLTP and OLAP data management (interactive queries and point updates) leading to reduction in complexity for both developers and ops. Additionally, SnappyData can dramatically reduce resource requirements (CPU, memory, network) through the use of probabilistic data: statistical techniques to maintain a small fraction of the data and respond to analytic questions at Google-like speeds. Using SnappyData is as simple as adding the SnappyContext to a tradition Spark program.

APIs

Developers can start working with SnappyData by installing and simply adding the SnappyContext in a Spark program. SnappyData exposes APIs for both streaming and reference data. Like Spark, no code changes are required between contexts. In both cases, client applications can submit Spark programs or connect using JDBC/ODBC and execute SQL queries.

  • SQL Queries & DML

    Spark-SQL forms the basis for queries on SnappyData. We extend Spark-SQL to support Update/Insert/Delete statements for transactional workloads.

  • Spark Applications

    Any Spark application can access SnappyData by simply adding the SnappyContext. This means MLib, PySpark, R and more can all be used with SnappyData.

  • Scala/Java

    Native Scala and Java applications can also access SnappyData

Queries

SnappyData bridges the OLTP/OLAP gap in data management by offering support for both analytical workloads and transactional workloads. OLAP queries (select …) are processed by the Catalyst engine and executed inside Apache Spark. OLTP queries (insert/update/delete) are executed inside the SnappyData in-memory database which is discussed below. A single application running on SnappyData can call both types of queries in streaming or batch contexts.

Apache Spark

Apache Spark is a fast general engine for large-scale data processing. Spark has been a runaway success in the business intelligenece/data science world. A big reason for Spark’s success is developer productivity: Spark abstracts away most of the complexity of distributed data processing and offers advantages like executing the same code in streaming and batch contexts. By integrating with Spark, SnappyData inherits all of the conveniences of Spark.

  • Integrates with Spark libraries

    Utilize all of Spark’s libraries like MLlib, PySpark and more

  • Spark abstractions are a first-class citizen

    Using SnappyData is as simple as creating and manipulating DataFrames, RDDs and other Spark abstractions.

The SnappyData Store

The Snappy-Store is a robust in-memory database. It is an extension of Pivotal’s GemFire which also goes by the open source name Apache Geode. This technology has been tested in production customer deployments for over a decade. The benefits of fusing Spark with an always-on, multi-tenanted database are manyfold. Firstly, it enables Spark to become a highly available, highly concurrent platform.

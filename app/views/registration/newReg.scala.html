Introducing Snappy

Big data analytics frameworks that used to require hundreds of servers, months of development and over $100k in costs can now be set up on 1/10th of the servers, at 1/10th of the cost and run 10x faster.

  • In-memory synopsis store

    Snappy is an in-memory, distributed store that creates synopsis data to deliver faster analytics queries on less hardware. Learn more on our blog

  • Spark & SQL interfaces

    Snappy aims for ease of use by exposing the well known Spark programming model and SQL query execution. Learn more on our blog

  • Streaming & Batch Architecture

    Use low latency approximate querying techniques in both batch and streaming contexts. Learn more on our blog

How does it work?

Snappy works by computing synopsis and probabilistic data structures that it maintains in memory. A Spark programming model is exposed to easily query Snappy's infrastructure. Learn more on our blog
